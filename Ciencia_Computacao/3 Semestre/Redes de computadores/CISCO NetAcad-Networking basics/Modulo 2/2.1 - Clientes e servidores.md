## 2.1.2 Funções de Clientes e Servidores

>[!info] Hosts na Rede
> - Todos os computadores conectados a uma rede que participam diretamente na comunicação são classificados como **[[Hosts]]**.
> - O *host* pode enviar e receber mensagens na rede.
> - Em redes modernas, um *host* pode atuar como **[[Cliente]]**, **[[Servidor]]** ou **ambos**. A função é determinada pelo software instalado.
### Servidores
- **Função:** São *hosts* que possuem software que lhes permite **fornecer informações e serviços** (como e-mail ou páginas Web) a outros *hosts* na rede.
- Cada serviço exige um software de servidor separado (ex: **Servidor Web**, **Servidor de E-mail**).
### Clientes
- **Função:** São *hosts* que possuem software que lhes permite **solicitar e exibir** as informações obtidas do servidor.
- **Exemplo de Software Cliente:** Navegador da Web (Chrome, Firefox, etc.).
### Tabela de Funções Cliente-Servidor

| Tipo de Serviço | Função do Servidor                       | Software Cliente Típico                |
| :-------------- | :--------------------------------------- | :------------------------------------- |
| **E-mail**      | Executa o software de servidor de e-mail | Microsoft Outlook, Mozilla Thunderbird |
| **Web**         | Executa o software do servidor web       | Navegadores da Web                     |
| **Arquivo**     | Armazena arquivos em local central       | Windows Explorer                       |

## 2.1.5 Múltiplas Funções na Rede

>[!tip] Flexibilidade de Hosts
> - Um único computador pode ter **múltiplas funções** na rede:
>   - **Múltiplas Funções de Servidor:** Um único computador pode executar software para atuar simultaneamente como **servidor de arquivos**, **servidor Web** e **servidor de e-mail** (comum em SOHOs ou empresas pequenas).
>   - **Múltiplas Funções de Cliente:** Um único cliente pode executar vários softwares cliente e se conectar a vários servidores ao mesmo tempo (ex: verificar e-mails, navegar na Web e ouvir rádio via Internet).

## 2.1.3 Redes Ponto-a-Ponto (P2P)

>[!note] Modelo P2P Simples
> - Ocorre quando softwares de cliente e servidor são executados no **mesmo computador** ou em **computadores que funcionam como ambos** na rede (cliente e servidor).
> - Redes P2P mais simples consistem em **dois computadores diretamente conectados** (com ou sem fio).
### 2.1.4 Aplicações Ponto-a-ponto (P2P)
- **Definição:** Uma aplicação **[[P2P]]** permite que um dispositivo atue como **cliente e servidor** na mesma comunicação. Nesse modelo, **todo cliente é um servidor e todo servidor é um cliente**.
- **Requisito:** Cada dispositivo final deve fornecer uma interface de usuário e executar um serviço em segundo plano.
- **Sistema Híbrido P2P:**
    - O compartilhamento de recursos é **descentralizado**.
    - Os **índices** (que apontam para as localizações dos recursos) são armazenados em um **diretório centralizado** (servidor de índice).
### Vantagens e Desvantagens das Redes P2P

| Vantagens                                                                          | Desvantagens                                                                                               |
| :--------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------- |
| **Fácil de configurar** e menos complexo.                                          | **Nenhuma administração centralizada** (dificulta o gerenciamento).                                        |
| **Menor custo**, pois servidores dedicados não são necessários.                    | **Não é tão segura**.                                                                                      |
| Pode ser usada para tarefas simples, como **compartilhar impressoras e arquivos**. | **Não é escalável** para grandes quantidades de usuários.                                                  |
|                                                                                    | O desempenho de um *host* pode ser **reduzido** se estiver atuando como cliente e servidor ao mesmo tempo. |
